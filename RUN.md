# Запуск бота напоминаний Discord

Краткая инструкция: что нужно подготовить и какие команды выполнить.

---

## Что должно быть установлено на компьютере

| Требование | Зачем |
|------------|--------|
| **Node.js 20+** | Среда выполнения JavaScript/TypeScript ([nodejs.org](https://nodejs.org/)) |
| **npm** | Идёт вместе с Node.js, ставит зависимости проекта |

Проверка в терминале:

```bash
node -v
npm -v
```

---

## Что настроить до первого запуска

### 1. Приложение и бот в Discord

1. Зайди в [Discord Developer Portal](https://discord.com/developers/applications) и создай приложение (или открой существующее).
2. Вкладка **Bot** — скопируй **токен бота** (не Public Key с вкладки «Общая информация»).
3. В **OAuth2 → URL Generator** (или **Installation**) отметь scope **`bot`** и **`applications.commands`**, выбери нужные **права бота** (минимум: отправка сообщений, при необходимости — упоминание @everyone, управление сообщениями для удаления своих сообщений).
4. Открой сгенерированную ссылку в браузере и **добавь бота на свой сервер**.

### 2. Файл `.env` в корне проекта

Скопируй `.env.example` в `.env` и заполни:

| Переменная | Откуда взять |
|------------|----------------|
| `BOT_TOKEN` | Developer Portal → **Bot** → Token |
| `GUILD_ID` | Discord: режим разработчика включён → ПКМ по **серверу** → «Копировать ID сервера» |
| `CHANNEL_ID` | ПКМ по **каналу** напоминаний → «Копировать ID канала» |

Файл `.env` не должен попадать в git (он уже в `.gitignore`).

### 3. Расписание напоминаний

Отредактируй **`config/reminders.json`**: время в cron, тексты, при необходимости `deleteAfterMinutes` у отдельных задач.

### 4. Права бота на сервере

Убедись, что в выбранном канале у роли бота есть права на отправку сообщений и (если используешь) на упоминание `@everyone`.

## Установка зависимостей (один раз)

В папке проекта (`discordBot`):

```bash
npm install
```

---

## Как запускать

### Режим разработки (удобно при правках кода)

Автоматически перезапускается при изменении файлов:

```bash
npm run dev
```

### «Продакшен» на своём ПК (без авто-перезапуска)

Сначала сборка TypeScript в `dist/`, затем запуск:

```bash
npm run build
npm start
```

Остановка: в терминале **Ctrl+C** (на Windows и macOS обычно так же).

---

## Что должно появиться в консоли при успехе

- Строка вида **`Бот запущен: ИмяБота#1234`**
- При наличии `GUILD_ID`: сообщение о регистрации slash-команд
- Строки **`[cron] Задача "…"`** по каждому напоминанию из `reminders.json`
- При наличии отложенных удалений: **`[pending-delete] Загружено отложенных удалений: N`**

Если бот не в сети — напоминания и отложенные удаления не выполняются: процесс должен быть **запущен** в момент срабатывания.

## Дополнительные файлы (создаются сами)

| Путь | Назначение |
|------|------------|
| `data/pending-deletions.json` | Очередь удаления сообщений (`/ping`, опросы) после перезапуска |

Папку `data` не нужно создавать вручную.

## Slash-команды

После запуска в канале сервера: **`/ping`**, **`/guild_bosses`** (если бот добавлен с scope `applications.commands` и `GUILD_ID` указан в `.env`).
