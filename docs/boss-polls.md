# Boss Polls — логика работы

## Расписание

| Время | Действие | Функция |
|---|---|---|
| Пн 09:00 МСК | Создать два опроса | `createBossPolls` |
| Чт 11:55 МСК | Прочитать voters пока опросы активны | `preReadBossPolls` |
| Чт 12:05 МСК | Опубликовать результаты | `publishBossResults` |
| Вс 09:00 МСК | Удалить сообщение с результатами | `cleanupBossResults` |

Код: `src/bossPolls.ts`. Крон-регистрация: `src/index.ts`.

---

## Понедельник 09:00 — создание опросов

Бот отправляет в `REMINDER_CHANNEL_ID` два сообщения с Discord-опросами:
- "На босса в четверг" (3 варианта времени, мультивыбор)
- "На босса в субботу" (2 варианта времени, мультивыбор)

Длительность: **75 часов** (истекают ровно в чт 12:00).  
ID сообщений и канала сохраняются в `data/boss-polls.json`.

---

## Четверг 11:55 — предварительное чтение голосов

За 5 минут до истечения опросов, пока они ещё **активны**:

1. Для каждого варианта обоих опросов вызывается `answer.voters.fetch({ limit: 100 })`
2. `voters.size` используется как счётчик голосов (на активном опросе `voteCount` недоступен)
3. Определяются победители (при ничье — несколько)
4. Для каждого победившего варианта формируется строка вида `**Четверг — 21.00-22.00:** @user1, @user2`
5. Все данные сохраняются в `boss-polls.json` в поле `preRead`

Именно этот шаг позволяет надёжно получить список проголосовавших: Discord не гарантирует работу `voters.fetch` на завершённых опросах.

---

## Четверг 12:05 — публикация результатов

**Основной путь** (preRead есть):
1. Читает готовые данные из `preRead` в `boss-polls.json`
2. Если есть голоса — отправляет постоянное сообщение в `VOTERS_CHANNEL_ID`:
   ```
   **Четверг — 21.00-22.00:** @user1, @user2
   **Суббота — 19.00-20.00:** @user3
   ```
3. Отправляет итоговое сообщение в `REMINDER_CHANNEL_ID`:
   - Есть победитель → `"По решению большинства голосов, поход на босса в __четверг__ - 21.00 - 22.00"`
   - Никто не голосовал → `"Поход на босса в __четверг__ отменяется, так как никто не хочет идти"`
4. Удаляет оба poll-сообщения, сохраняет `resultsMessageId`

**Fallback** (бот был офлайн в 11:55):
- Читает `voteCount` из завершённых опросов через `fetchFinalizedPollMessage` (retry до 10×3 сек)
- Отправляет только итоговое сообщение — без voters (список недоступен)

---

## Воскресенье 09:00 — очистка

1. Удаляет итоговое сообщение (`resultsMessageId`). Если уже удалено вручную — пропускает.
2. **Fallback:** пробует удалить poll-сообщения — на случай если четверговое удаление не сработало.
3. Обнуляет `boss-polls.json`.

Сообщение с voters в `VOTERS_CHANNEL_ID` **не удаляется** — оно постоянное.

---

## Startup reconciliation

При каждом запуске вызывается `runCleanupIfOverdue`.  
Если `resultsMessageId` задан **и** сейчас воскресенье или понедельник до 09:00 МСК — очистка запускается немедленно.

---

## Хранилище состояния

`data/boss-polls.json`:
```json
{
  "channelId": "...",
  "thursdayPollMessageId": "...",
  "saturdayPollMessageId": "...",
  "resultsMessageId": "...",
  "preRead": {
    "thursdayVoterLines": ["**Четверг — 21.00-22.00:** @user1"],
    "saturdayVoterLines": ["**Суббота — 19.00-20.00:** @user2"],
    "thursdayWinnerText": "21.00 - 22.00",
    "saturdayWinnerText": "19.00 - 20.00"
  }
}
```
`preRead` присутствует только между 11:55 и 12:05 чт; очищается после публикации.

## Каналы

| Переменная | Назначение |
|---|---|
| `REMINDER_CHANNEL_ID` | Опросы + итоговые сообщения (удаляются в вс) |
| `VOTERS_CHANNEL_ID` | Постоянный список проголосовавших |

## Тестовая среда

`src/bossPollsTest.ts` — изолированный модуль для тестирования полного цикла.  
Подключается вручную через `startBossPollTest` в `src/index.ts`.

Логика теста:
1. Создаётся опрос (duration=1ч)
2. За ~5с до принудительного завершения читаются voters через `answer.voters.fetch()`
3. `expirePoll` завершает опрос принудительно
4. Удаляется оригинальное poll-сообщение и системное сообщение Discord (тип 46)
5. Отправляются: постоянное сообщение со списком + временное с результатом (удаляется через 2 мин)
