## Приложение Discord

1. Зайди в [Discord Developer Portal](https://discord.com/developers/applications) → **New Application**.

2. Вкладка **Bot**:
   - Нажми **Reset Token** и скопируй токен → `BOT_TOKEN` в `.env`.
   - В разделе **Privileged Gateway Intents** включи **Server Members Intent** — без него событие «новый участник» не приходит.

3. Вкладка **OAuth2 → URL Generator**: выбери scope **`bot`** + **`applications.commands`**, затем права бота:

   | Право | Зачем |
   |-------|-------|
   | Send Messages | Напоминания, приветствия, опросы |
   | Read Message History | `/clear_survey` — сканирование истории |
   | Manage Messages | Удаление опросных сообщений и системных PollResult |
   | Mention Everyone | `@everyone` в напоминаниях |
   | Manage Roles | Выдача ролей Танк / Хиллер / Дамаггер / Друг гильдии |

4. Открой сгенерированную ссылку и добавь бота на сервер.

5. На сервере убедись, что роль бота в списке ролей **выше**, чем роли Танк / Хиллер / Дамаггер — иначе выдача не сработает.

## Переменные окружения (.env)

Файл `.env` не должен попадать в git (уже прописан в `.gitignore`).

Переменные без значения просто отключают соответствующую функцию — бот всё равно запустится.

### Обязательные

| Переменная | Где взять |
|------------|-----------|
| `BOT_TOKEN` | Developer Portal → Bot → Token |
| `GUILD_ID` | ПКМ по серверу → «Копировать ID сервера» |
| `REMINDER_CHANNEL_ID` | ПКМ по каналу напоминаний / опросов → «Копировать ID канала» |
| `WELCOME_CHANNEL_ID` | ПКМ по каналу приветствий → «Копировать ID канала» |

> **Режим разработчика** в Discord: Настройки → Расширенные → включить «Режим разработчика». После этого ПКМ по серверу или каналу покажет пункт «Копировать ID».

### Необязательные

| Переменная | Назначение |
|------------|-----------|
| `PROMO_CHANNEL_ID` | Канал для промо-кодов. Без него функция промо-кодов отключена |
| `VOTERS_CHANNEL_ID` | Канал для публикации списка проголосовавших после рейдовых опросов |
| `CLASS_LOG_CHANNEL_ID` | Канал лога классов (строки «ник — класс»). **Без него система классов отключена** |
| `ROLE_TANK_ID` | ID Discord-роли «Танк» |
| `ROLE_HEALER_ID` | ID Discord-роли «Хиллер» |
| `ROLE_DAMAGER_ID` | ID Discord-роли «Дамаггер» |
| `GUILD_FRIEND_ROLE_ID` | ID роли «Друг гильдии» (альтернатива игровым классам) |
| `CLASS_PROTECTED_ROLE_IDS` | ID ролей через запятую — модераторам/админам только обновляется строка в логе, игровые роли не трогаются |
| `CLASS_PROTECTED_USER_IDS` | ID пользователей через запятую (владелец сервера защищён автоматически) |
| `CLASS_STATS_SOURCE` | `json` (по умолчанию) или `roles` — источник подсчёта для сводки классов |
| `CLASS_STATS_EXCLUDE_BOTS` | `true` или `1` — не считать ботов в режиме `roles` |

## Конфиг напоминаний (`config/reminders.json`)

Файл перечитывается **автоматически** при изменении — перезапускать бота не нужно.

```json
{
  "timezone": "Europe/Moscow",
  "deleteAfterMinutes": null,
  "jobs": [
    {
      "id": "party-pre",
      "cron": "45 19 * * *",
      "message": "Через 15 минут начинается вечеринка гильдии!",
      "deleteAfterMinutes": 14
    },
    {
      "id": "party-start",
      "cron": "0 20 * * *",
      "message": "Вечеринка гильдии началась!",
      "deleteAfterMinutes": 60
    }
  ]
}
```

| Поле | Тип | Описание |
|------|-----|----------|
| `timezone` | string | Часовой пояс для всех задач (формат IANA, например `Europe/Moscow`, `UTC`) |
| `deleteAfterMinutes` | number \| null | Глобальный таймаут удаления. `null` — не удалять |
| `jobs[].id` | string | Идентификатор (только для логов, любая строка) |
| `jobs[].cron` | string | Cron-выражение из 5 полей: минута час день месяц день_недели |
| `jobs[].message` | string | Текст сообщения; отправляется с `@everyone` |
| `jobs[].deleteAfterMinutes` | number \| null | Переопределяет глобальный `deleteAfterMinutes` для этой задачи |

Сообщения, у которых `deleteAfterMinutes > 0`, удаляются через указанное время. Если бот перезапустился раньше — удаление из очереди восстанавливается при старте.

## Конфиг приветствия (`config/welcome.json`)

Файл также перечитывается при изменении без перезапуска.

```json
{
  "message": "Привет, {user}!\nДобро пожаловать на сервер **{guild}**."
}
```

| Плейсхолдер | Что подставляется |
|-------------|------------------|
| `{user}` | Упоминание нового участника |
| `{guild}` | Название сервера |

Если `message` пустое — приветствие не отправляется до следующей правки файла.

При входе **бота** отправляется шуточное сообщение вместо обычного приветствия.

## Система классов

Включается автоматически, когда в `.env` заданы все четыре переменные:  
`CLASS_LOG_CHANNEL_ID`, `ROLE_TANK_ID`, `ROLE_HEALER_ID`, `ROLE_DAMAGER_ID`.

### Сценарий приветствия

1. Новый участник заходит → в канале приветствия появляются кнопки **Танк / Хиллер / Дамаггер / Друг гильдии**.
2. Кнопка с кнопками **привязана к конкретному участнику** — чужую нажать нельзя.
3. После нажатия: выдаётся роль, в `CLASS_LOG_CHANNEL_ID` появляется строка `"ник" — "класс"`, сообщение с кнопками исчезает.

Если у нового участника уже есть любая небоевая роль на сервере — кнопки не показываются.

### Поведение лога

- Строка в логе **редактируется** при смене класса или ника, не дублируется.
- При выходе участника с сервера его строка **удаляется** из лога.
- При старте бот сверяет роли всех участников с логом и синхронизирует расхождения.

### Кнопка «Друг гильдии»

Выдаёт роль `GUILD_FRIEND_ROLE_ID` без записи в лог классов — для гостей или тех, кто не играет активно.

### Защищённые роли и пользователи

`CLASS_PROTECTED_ROLE_IDS` — ID ролей (через запятую). Участникам с такой ролью бот пишет строку в лог, но **не трогает** их игровые роли. Удобно для модераторов и администраторов.

Владелец сервера защищён автоматически.

### Сводка состава

В `CLASS_LOG_CHANNEL_ID` существует одно постоянное сообщение вида:

```
📊 Состав гильдии по классам

🛡️ Танк: 3
🌿 Хиллер: 2
⚔️ Дамаггер: 7
```

Обновляется автоматически после каждого выбора класса, смены ника и при старте.

**`CLASS_STATS_SOURCE`:**
- `json` (по умолчанию) — считает по записям из `config/class-user-messages.json` (только те, кто явно выбрал класс через бота).
- `roles` — считает участников с соответствующими ролями на сервере (учитывает и ручные назначения через Discord).

## Промо-коды

Включается, когда в `.env` задан `PROMO_CHANNEL_ID`.

### Как работает

1. Участник пишет сообщение в указанный канал — бот его удаляет.
2. Если код новый (не дубль) — бот публикует его **отдельным сообщением** от своего имени.
3. Каждая новая партия кодов — новое сообщение; старые сообщения не редактируются.

Это позволяет отслеживать «возраст» кодов по дате сообщения и автоматически удалять устаревшие.

### Автоочистка

- **При старте** бот удаляет собственные сообщения в канале, которым больше 30 дней.
- **Ежедневно в 04:00 МСК** запускается та же проверка.
- Закреплённые сообщения не удаляются.
- При удалении сообщения его коды возвращаются в «доступные» — те же коды можно прислать снова, и бот опубликует их как новые.

### Важно

- Бот работает **только с этим каналом** — сообщения в других каналах он не трогает.
- Канал промо-кодов лучше закрыть для обычной переписки; бот удаляет все не-закреплённые сообщения не от себя.

## Опросы боссов

Опросы публикуются и читаются из канала `REMINDER_CHANNEL_ID`.

### Расписание (МСК)

| День | Время | Действие |
|------|-------|----------|
| Понедельник | 09:00 | Публикуются два опроса: «На босса в четверг» и «На босса в субботу» |
| Четверг | 11:55 | Предварительное чтение голосов пока опросы ещё активны |
| Четверг | 12:05 | Публикация итогов, удаление опросных сообщений |
| Воскресенье | 09:00 | Удаление сообщения с результатами |

### Результаты

- Победивший вариант определяется по большинству голосов.
- Список проголосовавших с упоминаниями публикуется в `VOTERS_CHANNEL_ID` (если задан).
- Системные сообщения Discord о завершении опроса (тип 46) удаляются автоматически.

**Если бот был офлайн в 11:55** — при публикации в 12:05 используется fallback по счётчику голосов из завершённых опросов. Список проголосовавших в этом случае недоступен.

**Если бот пропустил воскресную очистку** — она выполнится при следующем старте.

**Если бот пропустил создание опросов в понедельник** — при следующем старте (с понедельника по среду включительно) опросы создадутся автоматически, если они ещё не существуют. Кроме того, ежедневно с понедельника по среду в 10:00 МСК бот проверяет наличие опросов и создаёт их, если они были пропущены.

## Slash-команды

Команды регистрируются на сервере (`GUILD_ID`) при каждом старте бота.

| Команда | Описание |
|---------|----------|
| `/ping` | Проверка доступности бота. Ответ удаляется через 15 минут |
| `/clear_survey` | Удалить все опросы бота в текущем канале и снять отложенные удаления |
| `/class <класс>` | Сменить основной класс вручную (Танк / Хиллер / Дамаггер). Работает так же, как кнопка при приветствии |

## Запуск

### Разработка

Автоматически перезапускается при изменении исходных файлов:

```bash
npm run dev
```

### Продакшен

```bash
npm run build   # компиляция TypeScript → dist/
npm start       # запуск из dist/
```

Остановка: **Ctrl+C** в терминале.

### Признаки успешного старта

```
Бот запущен: ИмяБота#1234
[slash] Команды зарегистрированы для сервера НазваниеСервера
[cron] Задача "party-start" → 0 20 * * * (Europe/Moscow)
[cron] Задача "boss-create" → 0 9 * * 1 (Europe/Moscow)
...
```

Если в `.env` заданы `CLASS_LOG_CHANNEL_ID` и ID ролей — дополнительно появятся строки `[class] Startup: сверяю роли N участников...`.

## Файлы состояния (создаются автоматически)

Не редактируй вручную. Папка `data/` создаётся при первом запуске.

| Путь | Содержимое |
|------|-----------|
| `data/pending-deletions.json` | Очередь отложенных удалений (сохраняется между перезапусками) |
| `data/boss-polls.json` | ID текущих опросов и результатов |
| `config/class-user-messages.json` | Карта userId → запись в лог-канале классов |
| `config/class-welcome-prompts.json` | Карта userId → сообщение с кнопками в канале приветствия |
| `config/class-stats-message.json` | ID постоянного сообщения сводки классов |
